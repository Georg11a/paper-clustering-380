Design-Knowledge Literature Review

Paper clustering pipeline for the final advancing-paper set

A reproducible workflow that groups papers by design-knowledge keyword, runs keyword-local clustering, flags small sample groups, and produces reviewable outputs for reading prioritization and citation-graph work.

View GitHub Repository Pipeline Overview

Input

Uses data/final_advancing_list.csv with paper metadata, abstracts, and keyword labels from the screening pipeline.

Method

Clusters papers separately within each keyword using k-means and HDBSCAN. DBSCAN remains available as an optional baseline.

Output

Writes per-paper cluster assignments, representative-paper rankings, summaries, and method-specific HTML explorers.

Run The Pipeline

Clone the repository, install dependencies, and run the keyword-conditioned clustering workflow.

python3 -m venv .venv
.venv/bin/python -m pip install -r requirements.txt
.venv/bin/python scripts/run_clustering.py \
  --input data/final_advancing_list.csv \
  --output outputs/clustering_results.csv

What The Batch Runner Produces

  • outputs/clustering_results.csv: combined per-paper results across keyword groups and methods.
  • outputs/keyword_summary.csv: keyword counts and cluster/skip status.
  • outputs/by_keyword/<keyword>/<method>/: method-specific CSV, UMAP page, explorer page, and cluster summary.

Small Keyword Groups

Keyword groups with fewer than 10 papers are not forced into clusters. They are kept in the combined output with cluster_status = too_few_papers, which makes the human checkpoint explicit.

Pipeline Flow

1

Load final screening CSV

Read paper_id, title, abstract, keyword, DOI, venue, URL, and screening rationale fields.

2

Group by keyword

Split the corpus into keyword-local subsets such as Design knowledge, Design theory, and Design patterns.

3

Apply small-sample rule

Groups with fewer than 10 papers are kept in the output but marked too_few_papers.

4

Cluster each keyword

Run k-means and HDBSCAN on title, abstract, keyword metadata, and selected full-text context when available.

5

Write review outputs

Generate combined CSV, keyword summary, cluster summaries, representative rankings, and HTML explorers.

Run Command

The main entry point is scripts/run_clustering.py.

python3 -m venv .venv
.venv/bin/python -m pip install -r requirements.txt
.venv/bin/python scripts/run_clustering.py \
  --input data/final_advancing_list.csv \
  --output outputs/clustering_results.csv

Output Interface

Output Purpose Status
outputs/clustering_results.csv Main downstream file with per-paper cluster labels, focus keyword, method, representative rank, and summary fields. Generated locally
outputs/keyword_summary.csv Per-keyword counts and whether each group was clustered or skipped. Generated locally
outputs/by_keyword/<keyword>/<method>/ Method-specific clustered CSV, UMAP visualization, explorer HTML, and cluster summary. Generated locally
cluster_status = too_few_papers Explicit marker for small keyword groups that should be reviewed rather than forced into clusters. Human checkpoint
